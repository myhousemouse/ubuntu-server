4. 디렉토리
    
    ```bash
    1 / 최상위 디렉토리
    2 /boot 부트디렉토리 -> 부팅시 커널이미지, 부팅정보
    3 /proc 현재 실행되는 프로세스, 장치 하드웨어
    4 /lib 공유 라이브러리 
    5 **/bin 기본적인 명령어 저장**
    6 /dev 시스템 디바이스, 하드디스크 CD
    **7 /etc 시스템 환경설정**
    **8 /root 시스템 관리자용**
    9 /sbin 관리자용 시스템 표준명령 
    10 /usr 사용자 디렉토리
    11 /home 사용자 계정 디렉토리, 로그인시 처음 위치 디렉토리
    **12 /var 가변자료저장, 로그파일, 메일 데이터 저장
    13 /temp 임시파일, 스티키 비트(모두가 읽고쓸수있지만, 파일 소유자만 소유파일을 지울수있음)**
    14 /mnt 파일 시스템 일시적 마운트
    15 /lost+found 결함이 있는 파일 정보 
    ```
    
5. 설치
    1. 미니멀,데스크탑,서버,랩탑,가상호스트,소스코드 컴파일,사용자설정 등 목적에 따라 설치
    2. 디스크 파티션 삭제 → 모든 파일 삭제되니 백업으로
    3. 부팅 드라이브 다시 파티션 → 부트로더 재설치, 운영체제 재설치 
    4. 하드웨어 호환성 확인
6. 파티션
    1. 파티션 : 물리적 디스크 → 여러 논리적 디스크로
        1. 주 파티션 - 부팅 가능 기본 파티션
            1. 하나의 하드디스크 → 최대 4개
        2. 확장 파티션 - 주 파티션 내에 생성
            1. 데이터 저장X 논리파티션 생성역할
        3. 논리 파티션 - 확장 파티션 안에 생성
            1. 12개 이상 생성시 권장X, 5번이후에 번호 생김
        4. 스왑 파티션 - 하드디스크 일부를 메모리처럼 사용
            1. 주 파티션 / 논리 파티션 내에 생성
            2. 메모리의 2배로 설정, 리눅스 내에 반드시 설치 됨
    2. 왜 ? → 관리 효율, 부팅 시간 줄이고 안정성 증가
    3. `/proc/partitions`
    4. fdisk - 파티션을 관리하는 명령어
    
    ```bash
    fdisk[장치명] == fdisk sdb
    ```
    
7. LVM (Logical Volume Manager)
    1. 다수 물리적 하드디스크 → 논리적 통합 → 하나의 파일 시스템으로 → 큰 용량 저장장치, 파티션 유용
    2. 물리볼륨 : 개별 물리적 디스크
    3. 볼륨 그룹: 여러개 물리 볼륨을 하나의 그룹으로
    4. 논리 볼륨: 볼륨 그룹의 일부를 논리그룹으로 사용
8. RAID(Redundant Array of Independent Disks)
    1. 여러개 물리적 디스크를 하나의 논리적 디스크로 구성 → 데이터 분할 저장, 중복 저장 → 성능향상
        1. RAID 0 : 스트라이핑 디스크에 나누어 저장 → 고용량 유리
        2. RAID 1 : 미러링, 중복저장 → 2배가 필요해짐, 결함 생겨도 괜찮
        3. RAID 0 + 1 : 스트라이핑 후 미러링 → 느린 복구, 빠른I/O
        4. RAID 1 + 0 : 미러링 후 스트라이핑 → 빠른 복구
9. 부트로더
    1. 운영체제를 주기억장치로, PC BIOS, OS Loader기능
    2. MBR(Master Boot Record), 하드디스크 첫번째 섹터에 저장됨
    3. LILO(Linux Loader) : 리눅스 전용
    4. GRUB2 : 편리한 설정, 대화형 UI,
10. 런레벨 
    1. `etc/inittab`에 설정 
        1. `id(코드/이름) : runlevel:action(프로세스 실행 지시어) : process(실행 프로세스) → runlevel명령`
    2. 서비스 실행을 0~6단계로 구성
    3. 런레벨 0 : 시스템 중지, 기본값 설정 불가 == init 0
    4. 런레벨 1 : 싱글 유저 모드, root로그인
    5. 런레벨 2 : 멀티 유저, 네트워크 사용안함
    6. 런레벨 3 : CLI, 다중사용자 모드
    7. 런레벨 4: 사용안함, 사용자 정의 모드
    8. 런레벨 5 : GUI 다중사용자, 최근 리눅스 기본값
    9. 런레벨 6: 리부트, 시스템 재부팅, 기본값 불가 == init 6
11. 로그인
    1. 로그인 메시지 설정 파일
        1. `/etc/issue` (로그인 전 콘솔 접속 시), `etc/issue.net` (로그인 전 원격 접속 시)
        2. `/etc/motd` (로그인 후 메시지)

→ 설정 값 적용은 `source`
